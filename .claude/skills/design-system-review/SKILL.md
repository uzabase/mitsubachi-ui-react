# デザインシステムレビュースキル

このスキルは、デザインシステムとReactに精通したフロントエンドエキスパートの視点で、mitsubachi-ui-reactプロジェクトのコードを厳しくレビューします。

## Use when:

- ユーザーが `/review` コマンドを実行した時
- コンポーネントの実装が完了し、レビューを求められた時
- プルリクエスト作成前のコードチェック時
- ユーザーが明示的にコードレビューを依頼した時

## レビュー観点

### 1. デザインシステムの使いやすさ
- **開発者体験（DX）**: このコンポーネントは他の開発者が直感的に使えるか？
- **API設計**: propsの命名や構造は分かりやすいか？必要最小限か？
- **デフォルト値**: 適切なデフォルト値が設定されているか？
- **ドキュメント**: TypeScriptの型定義やコメントで使い方が明確か？

### 2. 実装負担
- **学習コストの低さ**: 新しい開発者がすぐに使い始められるか？
- **設定の複雑さ**: 必須propsが多すぎないか？設定が煩雑ではないか？
- **カスタマイズ性**: 必要な時に拡張・カスタマイズできるか？
- **ボイラープレートコード**: 使用時に冗長なコードを書かせていないか？

### 3. コンポーネント設計
- **単一責任の原則**: コンポーネントの責務は明確か？
- **再利用性**: 異なるコンテキストでも使えるか？
- **composability**: 他のコンポーネントと組み合わせやすいか？
- **制約の適切さ**: 過度に制約的でも、過度に自由でもないか？

### 4. アクセシビリティ
- **セマンティックHTML**: 適切なHTML要素を使用しているか？
- **キーボード操作**: キーボードだけで操作できるか？
- **ARIA属性**: 必要なARIA属性が適切に設定されているか？
- **フォーカス管理**: フォーカス状態が適切に管理されているか？

### 5. パフォーマンス
- **不要な再レンダリング**: memo化やuseCallbackが適切に使われているか？
- **バンドルサイズ**: 不要な依存関係を含んでいないか？
- **レンダリングコスト**: 重い処理がレンダリング内で実行されていないか？

### 6. TypeScript型定義
- **型の正確性**: 型定義が実装と一致しているか？
- **型の厳密性**: anyやas使用を避けているか？
- **ジェネリクス**: 必要に応じてジェネリクスを活用しているか？

### 7. CSS設計（CLAUDE.mdに準拠）
- **class名セレクタ**: ID名セレクタやタグ名セレクタを使っていないか？
- **論理プロパティ**: 物理プロパティではなく論理プロパティを使用しているか？
- **擬似クラス**: hover、focus等は擬似クラスで実装されているか？
- **`:focus-visible`の使用**: `:focus`ではなく`:focus-visible`を使っているか？
- **レスポンシブ対応**: PCファーストで720px 1箇所のみのブレイクポイントか？

### 8. テスト容易性
- **テスタビリティ**: ユニットテストが書きやすい構造か？
- **data属性**: 必要に応じてtest-id等が設定されているか？

## レビュー手順

1. 変更されたファイルを特定する（git diffやgit statusを使用）
2. 各ファイルの内容を読み込む
3. 上記の観点で厳しくレビューする
4. 具体的な改善提案をコード例付きで示す
5. 良い点も積極的に指摘する

## 出力フォーマット

各ファイルごとに以下の形式でレビュー結果を出力してください：

```markdown
## 📄 ファイル名:行番号

### ⚠️ 問題点 / 💡 改善提案 / ✅ 良い点

**観点**: [該当する観点]

[具体的な指摘内容]

\`\`\`tsx
// ❌ 現在のコード
[問題のあるコード]

// ✅ 改善案
[改善されたコード]
\`\`\`

---
```

## チェックリスト

レビュー時は以下を確認:

- [ ] コンポーネントのAPI設計は直感的か
- [ ] 実装負担が少なく、使いやすいか
- [ ] propsの命名や型定義は適切か
- [ ] デフォルト値は合理的か
- [ ] セマンティックHTMLを使用しているか
- [ ] キーボード操作・ARIA属性は適切か
- [ ] CSSは論理プロパティを使用しているか
- [ ] `:focus-visible`を使用しているか（`:focus`は使っていないか）
- [ ] class名セレクタのみ使用しているか
- [ ] PCファーストで720pxのブレイクポイントか
- [ ] 不要な再レンダリングを避けているか
- [ ] anyや不適切なas使用を避けているか
- [ ] テストが書きやすい構造か
- [ ] Storybookストーリーは標準パターンに従っているか
