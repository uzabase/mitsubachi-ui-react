---

セッションコンテキストとGit履歴を活用して、コミット・プッシュ・プルリクエスト作成までの一連のワークフローを実行してください。

引数: $ARGUMENTS（コミットメッセージやPRに含めたい補足情報があれば使用する）

## 手順

### 1. 変更内容の把握

- `git status` で変更ファイルを確認する
- `git diff` でステージ済み・未ステージの差分を確認する
- セッション中の作業コンテキストも活用する

### 2. コミット

- CLAUDE.md のコミットルールに従ってコミットメッセージを作成する
- 変更ファイルをステージングし、コミットを実行する
- `$ARGUMENTS` に補足情報があればコミットメッセージに反映する

### 3. プッシュ

- 現在のブランチをリモートにプッシュする（`-u` フラグ付き）

### 4. PRの作成

- `git diff main...HEAD` と `git log main..HEAD` で全体の変更を把握する
- 以下のフォーマットに従ってPR本文を構成し、`gh pr create` で直接PRを作成する
- PRタイトルはCLAUDE.mdのコミットルールの形式に準拠する
- ベースブランチは `main` とする
- 作成したPRのURLを表示する

#### PR本文のフォーマット

```markdown
## 概要

## 変更内容

- hoge
- fuga

## 変更の影響

- 見た目の変更: 有り or 無し
- レビュワーに確認してほしいこと：有り or 無し

## 参考リンク

- Notion：https://www.notion.so/uzabase/2e158a33603580ecaf49c0b6e46e5e5a
- Figma：[変更したコンポーネント名]() // コンポーネントのFigma URL
```

#### 各セクションのルール

- **概要**: 必須。変更の目的と背景を簡潔に記載する
- **変更内容**: 必須。初見で理解できるよう具体的に書く
- **変更の影響**: プレースホルダーのみ記載する（手動で記入する）
- **参考リンク**: プレースホルダーのみ記載する（手動で記入する）
